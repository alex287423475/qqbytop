# 高考英语作文诊断工具上线运营与商业风控备忘录

> 本文档是 PRD 和 AI 辅助开发规划文档之外的运营补充，不改变主架构。  
> 目的：在开发进入 M1 后，同步锁定上线节奏、定价毛利、支付页表达、高考季扩容、AI 质量风控和淡季策略。

---

## 1. 定价与毛利测算

### 1.1 首版价格

首版只启用两个 `product_type`：

| 产品 | product_type | 价格 |
|---|---|---:|
| 最后冲刺抢分包（20 篇深度精诊额度） | `essay_credit_pack_20` | ￥99.00 |
| 三人同学组队抢分包（每人 20 篇额度） | `group_essay_credit_pack_20_member` | ￥53.00/人 |

权益说明：

- 每个订单获得 20 篇深度精诊额度，不是无限次，不是月卡，不按天/月重置。
- 从某个 `report_id` 进入付款时，支付成功立即消耗 1 篇额度解锁当前报告，剩余 19 篇可用于后续作文诊断。
- 三人同学组队价成团后，每名真实付费成员各自获得 20 篇额度；不共享作文、报告、订单或个人信息。
- `full_report_single` / `group_report_member` 仅作为旧链接兼容，不再作为前端主动售卖入口。

对应环境变量：

```text
GAOKAO_ESSAY_SINGLE_PRICE_CENTS=9900
GAOKAO_ESSAY_GROUP_PRICE_CENTS=5300
GAOKAO_ESSAY_PACK_CREDITS=20
GAOKAO_ESSAY_GROUP_REQUIRED_MEMBERS=3
GAOKAO_ESSAY_GROUP_EXPIRES_HOURS=24
GAOKAO_ESSAY_GROUP_ALLOW_PLATFORM_ASSIST=true
```

### 1.2 成本项

额度包订单应按最高 20 篇全部使用的压力场景核算，同时保留单篇实际使用成本监控：

| 成本项 | 说明 |
|---|---|
| OCR 成本 | 图片输入时产生；文本输入无 OCR 成本 |
| LLM 成本 | 诊断、范文、练习计划生成 |
| 对象存储 | 作文图片存储和读取 |
| PostgreSQL | 报告、订单、审计、归因数据 |
| Redis/队列 | OCR、诊断、退款、组队任务 |
| 支付手续费 | 微信/支付宝/服务商通道费 |
| 退款损耗 | 未出报告自动退款、支付手续费损耗 |

### 1.3 毛利底线

上线前必须用真实 provider 价格估算单次成本。保守底线：

- 独立抢分包 `￥99/20篇` 在 20 篇全部使用场景下仍需保持正毛利。
- 组队抢分包 `￥53/人/20篇` 在 20 篇全部使用场景下仍需覆盖 LLM、数据库、支付和退款损耗。
- 若免费诊断消耗超过预算，优先暂停免费新诊断，不影响已付费订单。

## 2. 支付页价值锚定

### 2.1 允许使用的表达

支付页可以强调真实产品差异：

- 按高考英语作文评分维度生成结构化诊断。
- 支持手写作文图片识别和校对。
- 解锁后包含逐句荧光笔标注、评分维度拆解、两版范文和 AI 演算母题库。
- AI 自动生成，不逐篇人工批改；订单异常支持自助重试/退款。

可使用的价值锚定文案：

```text
人工一对一作文批改通常需要更长等待时间。本工具提供 AI 自动生成的结构化诊断报告，适合考前快速发现问题。
```

避免写死无法证明的市场价格，如“人工批改通常 ¥80-200/篇”，除非页面能提供真实依据或改成“部分人工批改服务价格较高”。

### 2.2 禁止表达

- 不写“保分”“提分保证”“阅卷组内部”“名师亲批”。
- 不写“满分绝密”“致命丢分点”“不拉人就亏”。
- 不写“无条件秒退”“100% 消除投诉”。
- 不展示虚构同学、虚构头像、虚构班级群动态。
- 官方助力必须显示为官方助力名额，不得伪装真实用户。

## 3. 高考季扩容预案

### 3.1 时间表

| 时间 | 动作 |
|---|---|
| T-30 天 | 申请 OCR、LLM provider 提额；确认支付通道、退款通道和回调域名 |
| T-21 天 | 完成 30 篇高仿真作文质量抽检 |
| T-14 天 | 完成文本诊断、图片直传、OCR、支付、退款全链路压测 |
| T-7 天 | 检查监控、告警、队列阈值、免费额度、商户号状态 |
| T-1 天 | 冻结功能变更，只允许修复 P0/P1 故障 |

### 3.2 峰值降级策略

峰值时按以下顺序保护系统：

1. 暂停免费图片识别，保留文本免费摘要。
2. 暂停免费新诊断，已创建任务继续处理。
3. 付费任务进入 `default` 队列优先级前列。
4. 支付 webhook、订单同步、退款任务保持 `critical` 独立 worker。
5. LLM provider 拥塞时启用 fallback。
6. 队列超过硬阈值时返回排队提示或 429，不阻塞 webhook。

### 3.3 扩容触发条件

满足任一条件，进入人工扩容检查：

- 付费诊断 P95 完成时间超过 180 秒，持续 10 分钟。
- `critical` 队列积压超过 100 条。
- 支付成功未解锁订单超过 20 条。
- LLM provider 429/503 比例超过 20%，持续 5 分钟。
- Redis 内存或 CPU 超过 80%，持续 10 分钟。

## 4. AI 质量风控

### 4.1 上线前抽检

上线前至少检查 30 篇真实或高仿真高考英语作文，覆盖：

- 高分作文。
- 中等作文。
- 低分作文。
- 跑题作文。
- 明显语法错误作文。
- 图片 OCR 有误差的作文。

检查项：

- 预估分是否明显离谱。
- 维度评价是否与总分大体一致。
- 风险提示是否对应原文。
- 荧光笔位置是否错位。
- 范文是否跑题。
- 范文是否有明显语法错误。
- AI 演算母题库表述是否清晰、边界是否克制。

### 4.2 自动质量规则

后端可增加以下轻量后处理：

- `score.estimated` 不得超出 `0 <= score <= max_score`。
- `score.confidence = low` 时，前端应弱化确定性表达。
- 如果 `highlight_spans` 超过原文范围，必须降级为 `position_status = unresolved`。
- 如果范文字数明显偏离高考作文长度，应标记为需要重试或降级。
- 如果 LLM 输出缺少核心字段，必须 schema repair、fallback 或失败重试，不直接展示半成品。

### 4.3 无 RAG 报告质量闸门

首版不接 RAG、pgvector 或向量库，也不把真实学生作文沉淀为知识库资产。报告质量通过固定 Prompt 版本、严格 JSON Schema、后端质量校验和样例批量回归保证。

上线前质量闸门：

- 生产默认 Prompt 固定为 `backend/prompts/gaokao_default.md`；`gaokao_experiment.md` 只用于 A/B 批测，不直接上线。
- 每份报告必须写入 `prompt_version`、`provider`、`model_name`、`model_tier`，便于回溯质量问题。
- `FreeSummary` 和 `FullReport` 先通过 Pydantic Schema，再通过后端质量校验器。
- `free_summary.top_risks.type` 和 `highlight_spans.category` 必须使用生产 taxonomy 已知标签或精确中文等价表达；`misc_issue`、`语法问题` 这类泛化标签视为质量失败。
- 如果用户提供了作文题目或任务要求，完整报告必须体现审题、要点覆盖、格式、对象语气或跑题风险中的至少一类任务相关诊断信号。
- 如果报告使用分段校准标签，例如 `band_0_10_structure_collapse`、`band_19_21_plain_but_correct`，该标签必须与 `score.estimated` 所在分段一致。
- 质量校验失败必须触发重试或 fallback，不得展示半成品报告。
- 免费摘要不得泄漏原文片段、具体改写、范文、逻辑图或训练计划。

每次发生以下变更，必须重新跑 `backend/tools/batch_report_tester.py` 样例回归：

- 修改生产 Prompt。
- 更换默认模型或 provider。
- 修改诊断 JSON Schema。
- 修改报告渲染依赖字段。
- 修改禁用词、荧光笔定位或范文字数阈值。
- 修改 `backend/prompts/gaokao_writing_checklist.md`，尤其是稳妥版/进阶版范文生成规则。
- 修改 `backend/prompts/gaokao_error_taxonomy.md` 或后端 taxonomy 映射。
- 修改真题任务库、`task_context` 注入逻辑或题型识别规则。

默认批测合格线：

- Schema 通过率 100%。
- 单篇规则分不低于 80。
- 禁用词命中 0。
- Taxonomy 分类命中必须通过，不允许泛化分类占位。
- 带题目样例必须出现任务相关诊断信号。
- 分段校准标签不得与预估分冲突。
- 两版范文必须非空、贴合题型长度，并体现稳妥版与进阶版的层级差异。
- 每个失败样例必须输出明确原因，例如 `missing dimensions`、`banned terms found`、`highlight original not found`。

### 4.4 Provider 偏差监控

后台记录并观察：

- provider。
- model_version。
- prompt_version。
- score.estimated。
- max_score。
- latency_ms。
- fallback_path。

告警建议：

- 同类输入不同 provider 评分差异超过 5 分。
- 某 provider schema fail 超过 10%。
- 某 provider 平均分异常偏高或偏低。

## 5. 淡季策略

高考英语作文诊断具有明显季节性。淡季目标是降低成本、保留已付费报告访问能力，并为相邻考试场景复用做准备。

### 5.1 降本策略

- 降低 worker 数量。
- 暂停免费图片 OCR。
- 降低免费诊断全局额度。
- 保留报告查看、订单查询、退款查询和删除入口。
- 定期清理过期上传意图和已删除图片对象。

### 5.2 复用方向

后续可复用同一套上传、OCR、诊断、支付、退款和报告框架，但必须单独立项，不在本工具首版混入：

- 中考英语作文。
- 四六级英语写作。
- 考研英语作文。
- 高中英语月考/模考作文。

每个新场景必须单独定义评分维度、Prompt、文案和合规边界，不得把高考工具直接换皮上线。
