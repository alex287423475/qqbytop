## 1. 项目概述

### 1.1 一句话定义

海外商务第一印象诊断是一个面向跨境服务者、翻译公司、外贸顾问、出海创业者的在线诊断工具。用户上传一张商务头像或半身照，系统基于照片、身份、目标客户和使用场景生成商务形象诊断、标准报告、参考形象图、英文个人品牌文案，并把高意向用户导向付费在线包或人工定制方案。

### 1.2 产品定位

本项目不是颜值打分、AI 写真或泛娱乐头像工具，而是服务于海外商务沟通的信任表达工具。它评估的是照片在 LinkedIn、官网 About、展会、名片、邮件签名、商务资料页等触点里的专业度、可信度、跨文化表达和业务定位清晰度。

### 1.3 商业定位

这是一个诊断型获客产品，承担三类任务：

1. 免费教育：让用户意识到海外客户看到头像、简介和官网时会形成第一印象。
2. 需求分层：用基础诊断、套餐选择和定制表单判断用户付费意愿。
3. 服务转化：把有明确业务场景的用户导向英文个人品牌文案、商务形象包、翻译与合规沟通支持等人工服务。

### 1.4 可复用价值

该项目可作为其他“诊断工具型获客项目”的参考模板。通用结构是：

```text
免费基础诊断
-> 展示风险或改进空间
-> 低价在线报告
-> 专业包或高级报告
-> 人工定制服务
-> 线索通知和后续转化
```

该结构可以迁移到：

- Amazon Listing 翻译质量诊断
- 产品说明书出海合规检查
- 独立站英文转化问题诊断
- AI 翻译可用性评估
- 跨境合同翻译风险诊断
- LinkedIn 英文主页诊断
- 官网 About 页面可信度诊断

## 2. 背景与问题

### 2.1 用户背景

跨境服务者和出海企业常见问题是，业务能力真实存在，但海外触点缺乏统一表达：

- LinkedIn 头像像生活照、证件照或自拍照。
- 官网 About 页只有公司介绍，没有建立顾问可信度。
- 英文简介只是中文简历直译，没有说明服务对象、结果和专业边界。
- 展会名片、邮件签名、社媒头像、报价单风格不一致。
- 对海外客户而言，第一眼无法判断此人是否专业、可信、适合合作。

### 2.2 业务机会

全球博译的核心业务是翻译、本地化、跨境合规沟通。商务形象诊断虽然不是传统翻译服务，但它与英文个人品牌文案、官网 About 文案、跨境服务介绍、LinkedIn 资料改写天然相关，是一个低门槛、高感知、容易分享的前端入口。

### 2.3 当前实现基础

当前项目已经具备可用原型和部分生产能力：

- 主站工具矩阵已收录“海外商务第一印象诊断”。
- `/tools/business-image` 会重写或跳转到静态工具页面。
- 工具页包含上传照片、选择场景、基础诊断、套餐展示、样例报告、在线生成、下载和定制需求表单。
- 后端 API 支持基础诊断、标准报告、单张参考图重试、套餐生成和线索提交。
- 隐私政策、服务条款、照片与数据处理说明、关于页面已存在。
- AI 接口不可用时有 demo fallback，保证链路可演示。

### 2.4 当前不足

当前实现仍有若干产品化缺口：

- 工具主体是 `public` 下的静态 HTML/JS，不是完全 App Router 组件化实现。
- 线索提交目前主要依赖 Resend 邮件通知，没有稳定数据库、CRM 或飞书表格沉淀。
- 定制表单允许上传文件，但后端没有真正保存文件内容。
- 在线付费、订单、权益解锁、发票和退款流程尚未接入。
- 样例管理接口在生产环境禁用，当前更接近静态样例展示。
- 图片生成依赖 OpenAI 图像接口，可能存在超时、并发限制和不返回图片的情况。

## 3. 产品目标

### 3.1 MVP 目标

MVP 要完成以下闭环：

1. 用户能上传一张照片并选择使用场景。
2. 用户能看到免费基础诊断，包括评分、主要问题、推荐方向和 3 条建议。
3. 用户能理解付费报告和定制方案的差异。
4. 标准报告能在线生成 12 项建议和参考图提示或图片结果。
5. 专业形象包能生成英文个人品牌文案。
6. 定制方案能收集联系方式、业务身份、目标客户、使用场景、照片情况和文案需求。
7. 线索能通知运营者，便于人工跟进。

### 3.2 商业目标

- 将泛访客转化为可跟进线索。
- 通过低价报告验证付费意愿。
- 将高意向用户转化为定制形象方案、英文文案改写、官网 About 优化、跨境翻译和合规沟通服务。
- 为诊断工具矩阵积累可复制的产品、技术和运营模板。

### 3.3 用户价值目标

用户完成一次诊断后，应获得：

- 对当前商务头像主要问题的清晰判断。
- 针对目标客户和使用场景的优化方向。
- 可执行的照片、文案和渠道更新建议。
- 可直接复制或进一步人工修改的英文简介方向。
- 是否需要人工定制方案的判断依据。

## 4. 非目标

MVP 阶段不做以下内容：

- 不做颜值、年龄、健康、面相、性吸引力、种族、宗教、政治倾向等判断。
- 不承诺客户转化、平台审核、签约成功或收入提升。
- 不做完整 AI 写真工作室。
- 不开放多人团队协作后台。
- 不做复杂订单系统和会员系统。
- 不把用户照片公开展示为案例，除非另行获得明确授权。
- 不用 AI 直接替代人工高端定制建议。

## 5. 目标用户

### 5.1 核心用户

| 用户类型 | 典型身份 | 核心需求 | 付费可能性 |
| --- | --- | --- | --- |
| 跨境服务者 | 翻译顾问、合规顾问、外贸顾问 | LinkedIn、官网和邮件签名更专业 | 高 |
| 出海创业者 | 独立站创始人、SaaS 创始人、品牌负责人 | 面向海外客户建立可信创始人形象 | 高 |
| 外贸业务负责人 | 销售经理、展会负责人、业务负责人 | 展会资料、名片和客户会议形象统一 | 中高 |
| 自由职业者 | 翻译、设计、咨询、采购代理 | 用英文简介和头像提高咨询转化 | 中 |
| 小型服务公司 | 1-10 人团队 | 统一官网 About、顾问介绍和资料页 | 高 |

### 5.2 典型用户故事

1. 作为跨境合规翻译顾问，我希望知道当前头像是否适合 LinkedIn 和英文官网，以便让海外客户更容易信任我。
2. 作为出海创业者，我希望获得一套头像、英文简介和官网 About 的组合建议，以便更新对外资料。
3. 作为外贸展会负责人，我希望快速判断团队成员照片是否适合名片和展会资料，以便在展前统一商务形象。
4. 作为一人服务公司，我希望有一份可执行清单，以便在 30 天内更新 LinkedIn、官网、邮件签名和报价单。

## 6. 用户旅程

### 6.1 入口路径

用户可能从以下入口进入：

- 主站 `/zh/tools` 工具矩阵。
- 搜索引擎关键词，例如“LinkedIn 头像诊断”“英文个人简介优化”“海外商务形象”。
- 内容文章 CTA。
- 微信、朋友圈、社群或私域分享。
- 官网报价页或关于页内部推荐。

### 6.2 核心流程

```text
进入工具页
-> 阅读定位声明：不是颜值打分
-> 选择场景：LinkedIn / 出海创始人 / 展会洽谈 / 顾问官网
-> 上传头像或半身照
-> 填写行业身份和目标客户
-> 勾选目标：头像更专业 / 英文简介建议 / 展会或客户会议
-> 生成基础诊断
-> 查看评分、主要问题、推荐方向、3 条建议
-> 选择套餐
   -> 免费版：保留基础诊断
   -> 标准报告：生成 12 项报告 + 2 张参考图
   -> 专业形象包：补充业务信息，生成报告 + 6 个场景方案 + 英文简介
   -> 定制方案：提交联系方式和资料
-> 下载或提交需求
```

### 6.3 转化路径

| 用户行为 | 系统响应 | 业务意图 |
| --- | --- | --- |
| 只生成免费诊断 | 展示问题和套餐样例 | 教育用户 |
| 点击标准报告 | 在线生成低价报告 | 验证付费意愿 |
| 点击专业形象包 | 要求补充业务信息 | 获取更强意图数据 |
| 点击定制方案 | 展开线索表单 | 转人工服务 |
| 下载结果 | 鼓励保存和更新资料 | 增强工具价值 |
| 提交联系方式 | 邮件通知运营者 | 人工跟进成交 |

## 7. 功能需求

### 7.1 工具首页

#### 需求

- 页面首屏明确产品名称：海外商务第一印象诊断。
- 显示主站导航，保持与全球博译官网一致。
- 显示三步流程：上传头像、选择用途、生成报告。
- 明确声明“不是颜值打分”。
- 提供场景切换：LinkedIn 头像、出海创始人、展会洽谈、顾问官网。
- 页面应能在桌面和移动端使用。

#### 验收标准

- 用户 5 秒内能理解工具用途。
- 用户能在不登录的情况下开始上传。
- 敏感属性免责声明在诊断前可见。

### 7.2 图片上传与本地预览

#### 需求

- 支持 `image/*` 类型上传。
- 上传后在浏览器内显示预览。
- 前端压缩图片后再提交后端。
- 上传非图片文件时提示错误。
- 更换照片后重置已有报告结果。

#### 当前实现

`app.js` 中使用 `portraitInput` 监听上传，调用 `compressImageFile` 后保存为 `state.imageData`，再提交到 `/api/business-image/diagnose`。

#### 验收标准

- 上传照片后预览正常显示。
- 未上传照片时点击生成按钮，应提示先上传。
- 更换照片后旧报告不再显示为当前结果。

### 7.3 场景与目标输入

#### 需求

输入项包括：

- 使用场景：LinkedIn 头像、出海创始人、展会洽谈、顾问官网。
- 行业/身份：默认“跨境服务顾问”。
- 目标客户：默认“海外品牌方 / 外贸客户”。
- 目标勾选：头像更专业、英文简介建议、展会或客户会议。

#### 验收标准

- 不填写身份和目标客户时有合理默认值。
- 切换场景后报告重新生成，不复用旧场景结果。
- 所有输入会进入诊断 payload。

### 7.4 免费基础诊断

#### 需求

生成基础报告，包含：

- `score`：0-100 商务信任感或专业表达完整度评分。
- `issue`：主要问题。
- `direction`：推荐形象方向。
- `summary`：整体说明。
- `quickInsights`：正好 3 条建议。
- `paidModules`：12 项可升级模块。
- `source`：`openai` 或 `demo`。

#### 后端接口

```http
POST /api/business-image/diagnose
Content-Type: application/json
```

请求字段：

```json
{
  "scenario": "linkedin",
  "scenarioName": "LinkedIn 头像",
  "role": "跨境服务顾问",
  "audience": "海外品牌方 / 外贸客户",
  "goals": ["头像更专业", "英文简介建议"],
  "imageData": "data:image/jpeg;base64,..."
}
```

响应字段：

```json
{
  "score": 76,
  "issue": "照片中的商务定位信息还不够集中",
  "direction": "面向海外品牌方 / 外贸客户的清爽可信顾问型形象",
  "summary": "...",
  "quickInsights": ["...", "...", "..."],
  "paidModules": ["头像构图与裁切", "..."],
  "source": "openai",
  "context": {}
}
```

#### 验收标准

- OpenAI 可用时返回真实诊断。
- OpenAI 不可用时返回 demo fallback，页面不崩溃。
- 页面展示评分、主要问题、推荐方向和 3 条建议。
- Demo 结果必须明确标注。

### 7.5 标准报告

#### 需求

标准报告面向低价在线生成，包含：

- 12 项标准报告。
- 2 张参考形象图或参考图提示词。
- 可展示在线生成进度。
- 可展示部分完成状态。
- 可下载报告。

12 项模块固定为：

1. 头像构图与裁切
2. 表情与视线
3. 服装色彩
4. 背景与光线
5. 姿态与肩颈线条
6. 跨文化信任感
7. 行业身份匹配
8. LinkedIn 首屏表达
9. 英文个人简介
10. 官网 About 页面
11. 展会/名片场景
12. 30 天执行清单

#### 后端接口

```http
POST /api/business-image/generate-reference
Content-Type: application/json
```

`plan=standard` 时生成 12 项报告和 2 个参考图场景。

#### 验收标准

- 标准报告必须正好返回 12 项。
- 每项包含 `title`、`priority`、`advice`、`action`。
- 图片生成失败时仍显示提示词和错误原因。
- 用户能看到生成状态：完成、部分完成、图片未生成。

### 7.6 专业形象包

#### 需求

专业形象包需要先补充业务信息：

- 姓名或品牌名。
- 职位或身份。
- 主要服务。
- 核心优势。
- 希望客户采取的动作。

生成内容包括：

- 12 项标准报告。
- 6 个商务场景参考方案。
- 英文个人品牌文案。
- 可下载英文简介。

英文文案字段：

- LinkedIn Headline。
- LinkedIn About。
- Website About。
- Email Signature。

#### 参考图场景

1. LinkedIn 商务头像参考。
2. 官网顾问形象。
3. 展会洽谈形象。
4. 创始人介绍图。
5. 英文简介配图。
6. 商务社媒头像。

#### 验收标准

- 缺少姓名/品牌、身份、主要服务、核心优势时不能提交。
- 生成的英文文案不应包含中文、拼音、Markdown 或解释文字。
- 文案必须围绕服务对象、服务内容、优势和行动号召。
- 图片失败不能影响报告和英文文案展示。

### 7.7 单张参考图重试

#### 需求

当套餐生成时部分图片未返回，用户可对单张参考图点击重新生成。

#### 后端接口

```http
POST /api/business-image/generate-single-reference
Content-Type: application/json
```

#### 验收标准

- 同一时间只允许一个单张重试请求。
- 重试时显示进度和等待提示。
- 重试成功后更新当前卡片。
- 重试失败后保留错误提示，不影响其他结果。

### 7.8 定制方案线索表单

#### 需求

定制方案表单字段：

- 微信或手机号，必填。
- 当前身份/业务。
- 目标客户。
- 使用场景。
- 照片情况。
- 英文文案需求。
- 补充需求。
- 上传照片或资料。

#### 后端接口

```http
POST /api/business-image/leads
Content-Type: multipart/form-data
```

#### 当前实现

后端会校验 `contact`，生成 `id`。如果配置了 Resend，则将 payload 文本发送到 `LEAD_NOTIFY_EMAIL`。

#### 后续增强

- 保存线索到数据库或飞书多维表格。
- 保存文件到对象存储，表格只存文件链接。
- 给线索添加来源、套餐、场景、报告摘要和跟进状态。
- 支持自动发送用户确认邮件。

#### 验收标准

- 未填写联系方式时不能提交。
- 提交成功后显示明确成功信息。
- 邮件通知失败不应阻断用户提交。
- 上传文件在接入存储前不应误导用户已经被完整保存。

### 7.9 样例报告展示

#### 需求

页面需要展示完整付费样例，降低用户购买前的不确定性：

- 标准报告样例。
- 专业形象包样例。
- 定制方案样例。

#### 当前实现

静态样例和 JSON 样例同时存在：

- `public/tools/business-image/data/standard-report-sample.json`
- `public/tools/business-image/data/professional-package-sample.json`

生产环境禁用样例写入接口。

#### 验收标准

- 不依赖 AI 接口也能展示样例。
- 样例要明确说明实际交付包含什么。
- 样例不能伪装成用户当前照片的真实结果。

### 7.10 下载能力

#### 需求

支持下载：

- 全部结果。
- 标准报告。
- 英文简介。
- 已生成的参考图。

#### 验收标准

- 仅在有对应结果时显示下载按钮。
- 文件名可读，例如 `business-image-standard-report.txt`。
- 图片下载失败时不影响文本下载。

## 8. 后端与数据设计

### 8.1 环境变量

| 变量 | 用途 | 是否必填 |
| --- | --- | --- |

## 9. AI 与安全策略

### 9.1 AI 诊断边界

AI 只能基于可见、非敏感、商务场景相关信息输出建议：

- 构图。
- 光线。
- 背景。
- 服装色彩。
- 姿态。
- 表情自然度。
- 场景匹配。
- 英文资料表达。

不得输出：

- 年龄判断。
- 种族、民族、宗教、政治倾向。
- 健康、疾病、医学美容建议。
- 面相、命运、性吸引力。
- 颜值评分。
- 身份真实性判断。

### 9.2 Prompt 约束

基础诊断 prompt 必须包含：

```text
禁止推断或评价年龄、种族、民族、健康、面相、命运、性吸引力、宗教、政治倾向等敏感属性。
不要做医学美容、健康诊断或颜值打分。
评分只代表商务场景中的信任感和专业表达完整度。
```

### 9.3 图片生成边界

参考图生成应遵循：

- 保留本人真实面部特征。
- 不改变身份。
- 不做过度美化。
- 不生成虚假资质、证件、奖项或冒充场景。
- 不生成可能误导客户的职业身份。

### 9.4 失败兜底

AI 失败时必须：

- 返回 demo 或模板结果。
- 明确标注 `Demo`。
- 不把模板结果描述为真实 AI 分析。
- 不阻断用户继续了解样例或提交定制需求。

### 13.1 漏斗指标

| 生成 | 标准报告成功率、专业包成功率、图片生成成功率、图片部分完成率 |
| 下载 | 报告下载率、英文简介下载率、图片下载率 |
| 线索 | 表单提交率、有效联系方式率、人工跟进率 |
| 成交 | 报价率、成交率、客单价、退款率 |

### 13.2 产品健康指标

- AI 接口平均耗时。
- 图片接口平均耗时。
- 超时率。
- 前端 JS 错误率。
- API 5xx 率。
- 上传文件平均大小。
- 移动端完成率。

### 13.3 业务判断指标

重点看三项：

1. 免费诊断到定制表单提交率。
2. 专业形象包补充业务信息的完成率。
3. 人工定制线索的成交率和客单价。

如果标准报告点击率高但定制线索少，说明用户愿意自助但人工服务表达不够清楚。  
如果定制表单打开率高但提交率低，说明字段过多或信任不足。  
如果提交线索多但成交低，说明报告触发的是好奇心而不是明确服务需求。

## 14. 非功能需求

### 14.1 性能

- 首屏静态资源应尽量控制体积。
- 大样例 JSON 可延迟加载。
- 图片上传前必须压缩。
- 图片生成请求要有明确进度和超时提示。

### 14.2 可用性

- 无 API Key 时页面仍可演示。
- 图片生成失败时不影响文本报告。
- 用户不能因等待图片而丢失已生成文本。
- 移动端表单字段必须易填写。

### 14.3 安全

- 对用户输入做 HTML 转义。
- 后端 sanitize 文本字段。
- 上传文件类型和大小需要在前后端同时限制。
- 不在前端暴露 OpenAI、Resend、飞书等密钥。
- 不把用户照片写入日志。

### 14.4 合规

页面需要持续保留：

- 隐私政策。
- 服务条款。
- 照片与数据处理说明。
- 删除资料联系方式。
- 未成年人限制。
- 第三方 AI 处理说明。

## 18. 复用模板

其他诊断工具可以按以下结构复制。

### 18.1 复用字段

| 字段 | 示例 | 说明 |
| --- | --- | --- |
| `toolName` | 海外商务第一印象诊断 | 工具名称 |
| `domain` | 商务形象 | 诊断领域 |
| `inputType` | 图片 + 文本 | 输入类型 |
| `freeOutput` | 评分 + 主要问题 + 3 条建议 | 免费结果 |
| `paidModules` | 12 项报告模块 | 付费报告维度 |
| `upgradePlan` | 标准报告 / 专业包 / 定制方案 | 套餐结构 |
| `leadFields` | 联系方式、场景、资料情况 | 线索字段 |
| `humanService` | 人工定制形象方案 | 最终承接服务 |

### 18.2 复用流程

```text
定义用户场景
-> 定义免费诊断输入
-> 定义评分和三条建议
-> 定义 8-12 个付费模块
-> 定义样例报告
-> 定义低价包和专业包
-> 定义人工服务交付物
-> 定义线索字段
-> 定义隐私和免责声明
-> 接入通知和数据沉淀
```

### 18.3 复用原则

1. 免费诊断只揭示问题，不交付完整解决方案。
2. 低价报告交付结构化建议，不能制造过高人工成本。
3. 高价方案必须有明确文件交付物。
4. AI 作为生成和分层工具，人工服务作为最终可信交付。
5. 所有诊断工具都要有样例报告，否则用户很难理解付费价值。
6. 表单字段越靠近真实交付，线索质量越高。
7. 不要把诊断工具做成泛聊天窗口，要让用户在一次流程中得到明确结果。

## 19. 对其他项目的参考结论

海外商务第一印象诊断最值得复用的不是“头像诊断”本身，而是以下产品结构：

- 一个低门槛输入。
- 一个即时反馈结果。
- 一个明确的风险或改进空间。
- 一个可预览的付费报告样例。
- 一个低价自动化包。
- 一个高价人工定制入口。
- 一个能被运营者跟进的线索数据结构。

对于翻译和跨境合规业务，后续所有诊断工具都应尽量遵循这个结构。这样每个工具既能独立获取搜索流量，也能统一导向人工翻译、合规审校、本地化改写和顾问服务，避免做成无法成交的免费玩具。
