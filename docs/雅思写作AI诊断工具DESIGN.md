# 雅思写作 AI 诊断工具 DESIGN

## 设计定位

本工具不是教培营销落地页，而是一个面向考生的写作诊断工作台。第一屏必须让用户立即粘贴作文并看到“免费基础报告”的价值，不做大面积空泛 hero，不使用夸张插画和单一紫蓝渐变。

核心感受：可信、清晰、可执行。页面像“考试诊断 cockpit”，而不是聊天机器人。

## 参考综合

- Linear 式的密集产品界面：用于报告页的信息层级、状态卡片和细边框。
- Stripe 式的交易可信感：用于套餐页、点数和支付前说明。
- Mintlify 式的文档化可读性：用于题库页和诊断说明，强调结构化和可扫描。

这些参考只用于抽象设计原则，不复制品牌配色、排版或组件。

## 视觉原则

- 主背景使用近白色和浅冷灰，保留考试工具的安静感。
- 关键面板使用深墨蓝，但只作为信息焦点，不让整站变成深蓝主题。
- 绿色用于“可执行建议”和已通过状态，红色只用于错误证据，琥珀色用于风险和待解锁提示。
- 卡片圆角不超过 8px，避免过度圆润的消费娱乐感。
- 页面 section 是全宽布局内的功能分区，不做卡片套卡片。
- 文本字号按工具界面密度控制，不用超大营销标题占满首屏。

## 页面结构

### `/tools/ielts-writing`

目标：无登录提交作文，进入免费基础报告。

模块：
- 顶部产品导航：题库、套餐、我的报告。
- 首屏价值说明：30 秒定位扣分点，明确基础报告免费、深度报告付费。
- 作文输入面板：Task 1 / Task 2、题目、正文、字数计数、提交按钮。
- 右侧说明：基础报告和深度报告区别、服务端权限底线。
- 题库导流区：展示 3 个高频题目，承接 SEO 流量。

### `/tools/ielts-writing/report/[reportId]`

目标：基础报告建立信任，深度报告制造解锁动机。

模块：
- 报告头：reportId、题目、分数区间、字数。
- 四项雷达图：TR / CC / LR / GRA。
- 维度卡片：每项分数、摘要和原文证据。
- Top Issues：最影响提分的 3 个问题。
- Sample Revision：免费展示 1 句修改。
- Deep report locked：深度报告锁定，跳转套餐页。

### `/tools/ielts-writing/pricing`

目标：强调写作点数独立，降低套餐理解成本。

套餐：
- ￥9.9：单篇解锁。
- ￥39：5 次写作深度诊断，主推。
- ￥99：15 次冲刺包。

所有套餐只增加 `writing_deep_credits`，不与 `speaking_deep_credits` 共享。

### `/tools/ielts-writing/profile`

目标：承接手机号登录后的复购和复盘。

模块：
- 写作点数余额。
- 口语点数展示为独立资产，防止混淆。
- 最近分数趋势。
- 历史报告列表。

### `/tools/ielts-writing/topics`

目标：SEO 题库流量入口。

要求：
- 每个题目页必须有真实写作帮助，不做空洞关键词页。
- 报告页 noindex，题库页 index。

## 组件约束

- CSS 使用 `.iw-*` 命名空间，避免污染现有口语工具的 `.ielts-*` 样式。
- V1 不引入新的 UI/图表依赖，雷达图用轻量 SVG 实现。
- 所有 mock 数据集中放在 `lib/ielts-writing/constants.ts`，便于后续替换为 API 返回。
- 深度报告锁定区不能包含真实深度报告内容，后续接 API 时也必须由服务端不返回 `deep_result_json`。
